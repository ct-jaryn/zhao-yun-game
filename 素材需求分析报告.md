## 4. 投射物与特效素材分析

### 4.1 Projectile

文件：`src/phaser/entities/Projectile.js`

当前实现：所有 Projectile 均为 `Graphics.fillRect` 绘制的纯色长条。

代码片段：
```js
const graphics = this.scene.add.graphics();
graphics.fillStyle(parseInt(this.color.replace('#', '0x'), 16) || 0xff8800, 1);
graphics.fillRect(-this.size * 1.5, -this.size / 2, this.size * 3, this.size);
this.sprite = this.scene.add.container(this.x, this.y, [graphics]);
```

虽然 `Projectile` 支持 `imgKey` 参数，但所有调用都传入 `null`：
- 玩家大招 I：`game.addProjectile(..., null, true)`
- 弓箭手：`new Projectile(...)` 未使用图片

| 使用方 | 当前颜色/形状 | 建议素材 |
|---|---|---|
| 玩家大招金色飞射物 | 金色长条 | 龙胆枪气/金色流光 |
| 弓箭手箭矢 | 橙色长条 | 箭矢/羽箭 |
| 可能 future 的 boss 远程 | 按颜色长条 | 飞斧/锤气/暗器 |

### 4.2 粒子系统

文件：`src/phaser/entities/Particle.js`

当前实现：每个粒子是 `Graphics.fillRect` 绘制的纯色方块，无纹理、无旋转、无混合。

```js
this.graphics.fillStyle(hex, 1);
this.graphics.fillRect(-this.size / 2, -this.size / 2, this.size, this.size);
```

建议素材（可复用 Phaser 粒子纹理或少量贴图）：

| 使用场景 | 建议纹理 |
|---|---|
| 受击/击杀 | 血雾、火星、碎屑 |
| 升级 | 金色星尘、光斑 |
| 技能释放 | 火焰、旋风、闪电碎片 |
| 闪避 | 蓝色残影、风痕 |
| Boss 死亡 | 爆炸烟雾、紫色能量 |

> 当前已用对象池优化性能，但视觉表现仍有较大提升空间。

### 4.3 屏幕特效

当前使用 Phaser 原生 `camera.shake` / `camera.flash`，无需额外素材。

## 9. 结论

当前项目的美术资源覆盖率已经较高，核心角色动画、章节背景、装备图标、技能图标等均有专属素材。主要素材缺口集中在：

1. **特效与 Projectile**：大量依赖 `Graphics` 纯色图形，是最明显的"廉价感"来源
2. **敌人死亡/受击**：缺少独立动画，靠代码变形弥补
3. **已存在但未加载的资源**：典韦/吕布/许褚的 `skill_frames` 被遗忘
4. **小地图与方向提示**：功能性完整但视觉简陋
5. **玩家 U 技能**：复用大招帧，辨识度不足

如果按优先级补充，建议先做 **Projectile 纹理 + 粒子纹理 + 敌人死亡帧 + 典韦 ultimate 加载**，这四项投入产出比最高，能显著提升游戏打击感和视觉品质。
